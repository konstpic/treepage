# Первый вход

## Локальный администратор (первый запуск)

При первом запуске TreePage автоматически создаёт учётную запись администратора и **welcome-пространство** с документацией.

### Учётные данные

| Поле | Значение |
|------|----------|
| Email | `admin@local` |
| Пароль | `admin` |
| Роль | `super_admin` |

> **Всегда доступен** — для первичной настройки и когда OIDC-провайдер недоступен. **Сразу после входа смените пароль:** Настройки → Пользователи.

### Пошагово

1. Запустите `docker compose up --build` (или `./scripts/deploy-dev.sh`)
2. Откройте http://localhost:8080
3. Нажмите **Войти** → `/auth`
4. Введите `admin@local` / `admin` (поля могут быть предзаполнены)
5. После входа откройте **Пространства** → **Welcome** (`/spaces/welcome`)

### Приветствие и обучение

После успешного входа (локально или через OIDC):

1. **Splash-экран** — анимация логотипа; может отображаться ваше имя
2. **Тур по интерфейсу** — краткое обучение навигации и разделам (после splash; можно пропустить или запустить снова через иконку 🎓)

Подробнее: [Обучение по интерфейсу](../user/onboarding-tour.md)

О welcome-пространстве: [Welcome space](welcome-space.md).

## Локальный админ и OIDC

На странице входа всегда есть **локальный вход** (для учёток с паролем) и **OIDC**, если он настроен. Локальный admin — для первичной настройки; OIDC — для повседневного SSO.

Тест OIDC с Authentik локально: [Тест OIDC с Authentik](authentik-oidc-test.md).

## Production: вход через OIDC

В production используется SSO через OIDC-провайдер (Keycloak, Okta, Azure AD и др.).

### Как войти

1. Откройте URL вашего TreePage (например `https://docs.example.com`)
2. Нажмите **Войти** → **Продолжить с OIDC**
3. Вас перенаправит на страницу провайдера идентификации
4. После успешной аутентификации вы вернётесь в TreePage с активной сессией

### Что происходит при первом OIDC-входе

1. TreePage получает authorization code от OIDC-провайдера
2. Обменивает code на токены
3. Создаёт или обновляет учётную запись пользователя в БД
4. Сопоставляет роли и группы из claims OIDC
5. Выдаёт JWT access/refresh токены
6. Перенаправляет на `/auth/callback` → `/spaces`

## Сессия и токены

| Параметр | Значение по умолчанию |
|----------|----------------------|
| Access token TTL | 15 минут |
| Refresh token TTL | 7 дней |

Frontend автоматически обновляет access token при истечении. При выходе refresh token отзывается.

## Публичный доступ

Некоторые пространства могут быть помечены как **публичные** — их документация доступна без входа. **Welcome**-пространство публичное по умолчанию. Кнопка **Открыть пространства** на главной странице работает без авторизации.

## Проблемы со входом

| Симптом | Решение |
|---------|---------|
| «OIDC недоступен» | Проверьте настройки OIDC в Helm/config и доступность провайдера |
| Redirect loop | Проверьте `frontendUrl` и redirect URL в OIDC-клиенте |
| «Не удалось войти» (локально) | Проверьте email/пароль; у учётки должен быть локальный пароль |
| 401 на API после входа | Проверьте совпадение `JWT_SECRET` между auth и server |

Подробнее: [Устранение неполадок](../operator/troubleshooting.md).
