# Первоначальная настройка

Пошаговое руководство после первого входа под локальным администратором `admin@local` / `admin`.

> Welcome-пространство (`/spaces/welcome`) создаётся автоматически — см. [Welcome space](welcome-space.md).

## Шаг 0. Welcome space

После первого запуска уже доступно:

- **Пространство:** Welcome (`/spaces/welcome`)
- **Источник:** [github.com/konstpic/treepage](https://github.com/konstpic/treepage) → папка `docs/`
- **Доступ:** публичный (можно читать без входа)

Если документов нет — нажмите **Синхронизировать** в sidebar (нужен push в GitHub с папкой `docs/`).

---

## Шаг 1. Системные настройки

Перейдите: **Настройки** → **Настройки системы** (`/admin/settings`)

### Язык интерфейса

Выберите **Русский** или **English**. Применяется глобально для всех пользователей.

### Тема оформления

| Тема | Описание |
|------|----------|
| Fox White | Светлая тема по умолчанию |
| Coral Night | Тёмная с акцентами |
| Light | Классическая светлая |
| Dark | Классическая тёмная |

### Аутентификация

| Параметр | Рекомендация (production) |
|----------|--------------------------|
| Включить OIDC | ✅ Да |
| Разрешить локальный вход | ❌ Нет (только dev) |

### Git-интеграция (глобальные defaults)

| Параметр | Значение по умолчанию | Описание |
|----------|----------------------|----------|
| Глобальная ссылка на токен | `GIT_ACCESS_TOKEN` | Имя env-переменной с Git-токеном |
| Webhook secret ref | `GIT_WEBHOOK_SECRET` | Имя env-переменной с секретом webhook |
| Интервал синхронизации | `300` сек | Default для новых репозиториев |
| Режим синхронизации | `scheduled` | manual / scheduled / webhook |

### Платформа

| Параметр | По умолчанию |
|----------|-------------|
| Лимит поиска | 20 |
| Макс. лимит поиска | 100 |
| Кэш | выключен |
| Уровень логирования | info |
| Автоперевод документов | выключен (требует LLM) |

Нажмите **Сохранить настройки**.

## Шаг 2. Настройка OIDC (production)

Перейдите: **Настройки** → **OIDC-провайдеры** (`/admin/oidc`)

> Доступно только для `super_admin`.

### Создание провайдера

| Поле | Пример |
|------|--------|
| Name | `Keycloak` |
| Issuer URL | `https://keycloak.example.com/realms/treepage` |
| Client ID | `treepage` |
| Redirect URL | `https://docs.example.com/api/auth/callback` |
| Scopes | `openid profile email` |
| Role claim | `roles` |
| Group claim | `groups` |
| Sync groups | ✅ |

### Настройка на стороне OIDC-провайдера

1. Создайте клиент типа **confidential** или **public** (с PKCE)
2. Укажите redirect URI: `https://<ваш-домен>/api/auth/callback`
3. Добавьте scopes: `openid`, `profile`, `email`
4. Сохраните client secret в Kubernetes Secret (`oidc-client-secret`)

Подробнее: [OIDC-провайдеры](../admin/oidc.md)

---

## Шаг 3. Создание пространства

Перейдите: **Настройки** → **Пространства** (`/admin/spaces`)

1. Заполните форму:
   - **Slug** — URL-идентификатор (например `engineering`)
   - **Название** — отображаемое имя (например `Engineering Docs`)
   - **Описание** — краткое описание (необязательно)
   - **Публичный** — включите, если документация должна быть доступна без входа
2. Нажмите **Создать**

Подробнее: [Пространства (admin)](../admin/spaces.md)

---

## Шаг 4. Подключение Git-репозитория

Перейдите: **Настройки** → **Репозитории** (`/admin/repositories`)

1. Нажмите **Добавить репозиторий**
2. Заполните:

| Поле | Пример |
|------|--------|
| Пространство | `Engineering Docs` |
| Название | `Main Docs Repo` |
| URL | `https://github.com/org/docs.git` |
| Ветка | `main` |
| Путь к документам | `docs` |
| Режим синхронизации | `scheduled` |
| Интервал (сек) | `300` |
| Токен доступа | `GIT_ACCESS_TOKEN` (или literal token) |
| Включён | ✅ |

3. Сохраните

### Структура репозитория

TreePage сканирует `{docs_path}/**/*.md`. Пример:

```
docs/
├── README.md           → slug: readme
├── guides/
│   ├── installation.md → slug: guides/installation
│   └── first-steps.md  → slug: guides/first-steps
└── api/
    └── reference.md    → slug: api/reference
```

Заголовок страницы берётся из первого `# H1` или из имени файла.

Опционально — теги через frontmatter:

```markdown
tags: kubernetes, helm, deployment

# Заголовок документа
```

Подробнее: [Репозитории](../admin/repositories.md), [Git Sync](../admin/git-sync.md)

## Шаг 5. Первая синхронизация

### Из панели администратора

1. **Настройки** → **Репозитории**
2. Найдите репозиторий → нажмите **Синхронизировать** (иконка обновления)

### Из интерфейса пространства

1. Откройте `/spaces/<slug>`
2. В боковой панели нажмите **Синхронизировать**

### Проверка

- Статус синхронизации: `success` / `completed`
- Документы появляются в дереве слева
- При ошибке — проверьте `last_sync_error` в списке репозиториев

## Шаг 6. Настройка пользователей и доступа

### Создание пользователей

**Настройки** → **Пользователи** (`/admin/users`)

Создайте учётные записи или дождитесь автоматического создания через OIDC.

### Группы

**Настройки** → **Группы** (`/admin/groups`)

Создайте группы (например `developers`, `ops-team`) для массового назначения прав.

### Права на пространство

**Настройки** → **Пространства** → редактирование пространства:

- **Участники** — назначьте пользователей с ролью `viewer`, `editor` или `admin`
- **Группы** — назначьте группы с ролью

Подробнее: [RBAC](../admin/rbac.md), [Пользователи](../admin/users.md), [Группы](../admin/groups.md)

---

## Шаг 7. Проверка

1. Откройте `/spaces/<slug>` — документы отображаются
2. Попробуйте **Поиск** (`/search?q=...`)
3. Войдите под пользователем с ролью `viewer` — проверьте доступ
4. Войдите под `editor` — проверьте редактирование
