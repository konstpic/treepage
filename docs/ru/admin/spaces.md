# Управление пространствами

**URL:** `/admin/spaces`

## Создание пространства

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Slug | ✅ | URL-идентификатор (латиница, дефисы). Пример: `engineering` |
| Название | ✅ | Отображаемое имя |
| Описание | ❌ | Краткое описание |
| Публичный | ❌ | Доступ без авторизации |

После создания пространство доступно по `/spaces/{slug}`.

## Редактирование

Нажмите иконку карандаша на карточке пространства.

### Основные поля

- Название, описание, флаг публичности

### Привязанные репозитории

В режиме редактирования отображаются Git-репозитории, связанные с пространством.

- **Отвязать** — удалить связь (репозиторий остаётся в системе)
- Для привязки нового репозитория — создайте его в **Репозитории** с выбором этого пространства

### Участники

| Поле | Описание |
|------|----------|
| Пользователь | Выбор из списка |
| Роль | `viewer`, `editor`, `admin` |

Добавление: выберите пользователя → роль → **Добавить**.

### Группы

| Поле | Описание |
|------|----------|
| Группа | Выбор из списка групп |
| Роль | `viewer`, `editor`, `admin` |

Все участники группы получают указанную роль в пространстве.

## Удаление

Удаление пространства удаляет связанные документы из TreePage (не из Git). Операция необратима.

## Slug: правила

- Только латиница, цифры, дефисы
- Уникален в системе
- Используется в URL: `/spaces/{slug}`
- После создания slug изменить нельзя — создайте новое пространство

## Примеры конфигурации

### Публичная документация продукта

```
Slug: product-docs
Публичный: ✅
Участники: не требуются для чтения
```

### Внутренняя документация команды

```
Slug: platform-team
Публичный: ❌
Группы: platform-engineers → editor
        platform-leads → admin
```
