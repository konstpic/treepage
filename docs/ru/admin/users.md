# Управление пользователями

**URL:** `/admin/users`

> Создание пользователей — только `super_admin`. Редактирование viewer/editor — `admin` и `super_admin`.

## Список пользователей

Отображает всех пользователей с:

- Email
- Отображаемое имя
- Роли
- Статус (активен / неактивен)

## Создание пользователя (super_admin)

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Email | ✅ | Уникальный email |
| Пароль | ✅ | Для локального входа |
| Отображаемое имя | ❌ | Имя в UI |
| Роли | ✅ | Одна или несколько |
| Активная учётная запись | ❌ | ✅ по умолчанию |

### Доступные роли

| Роль | Описание |
|------|----------|
| super_admin | Полный контроль |
| admin | Управление пространствами |
| editor | Редактирование |
| viewer | Чтение |

## Редактирование

Нажмите **Изменить** на карточке пользователя.

| Поле | Описание |
|------|----------|
| Email | Изменение email |
| Новый пароль | Оставьте пустым, чтобы не менять |
| Отображаемое имя | Имя в UI |
| Роли | Назначение ролей |
| Активная учётная запись | Деактивация блокирует вход |

## Ограничения для admin

Пользователь с ролью `admin` (не super_admin):

| Действие | Разрешено |
|----------|-----------|
| Просмотр всех пользователей | ✅ |
| Создание пользователей | ❌ |
| Редактирование viewer/editor | ✅ |
| Редактирование admin/super_admin | ❌ |
| Назначение admin/super_admin | ❌ |
| Удаление admin/super_admin | ❌ |

## Удаление

- Нельзя удалить свою учётную запись
- Нельзя удалить последнего super_admin
- Admin не может удалять admin/super_admin

## OIDC-provisioning

При входе через OIDC пользователи создаются автоматически:

1. Email из claim `email`
2. Display name из `name` или `preferred_username`
3. Роли из claim `roles` (настраиваемый)
4. Группы из claim `groups` (если `sync_groups: true`)

Ручное создание нужно только для локального входа (dev) или сервисных аккаунтов.

## Dev-аккаунт по умолчанию

При `DEV_MODE=true` bootstrap создаёт:

| Email | Пароль | Роль |
|-------|--------|------|
| admin@local | admin | super_admin |

> Смените пароль после первого входа.
