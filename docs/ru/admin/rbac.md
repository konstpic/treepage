# Роли и права (RBAC)

TreePage использует двухуровневую модель доступа: **системные роли** и **роли в пространстве**.

## Системные роли

Назначаются глобально через **Пользователи** (`/admin/users`).

| Роль | Описание |
|------|----------|
| `super_admin` | Полный контроль платформы |
| `admin` | Управление пространствами, репозиториями, группами |
| `editor` | Редактирование документов, sync, книги |
| `viewer` | Только чтение |

### Матрица системных прав

| Действие | super_admin | admin | editor | viewer |
|----------|:-----------:|:-----:|:------:|:------:|
| Системные настройки (запись) | ✅ | ❌ | ❌ | ❌ |
| OIDC-провайдеры | ✅ | ❌ | ❌ | ❌ |
| CRUD пользователей | ✅ | частично | ❌ | ❌ |
| CRUD групп | ✅ | ✅ | ❌ | ❌ |
| CRUD пространств | ✅ | ✅ | ❌ | ❌ |
| CRUD репозиториев | ✅ | ✅ | ❌ | ❌ |
| Редактирование документов | ✅ | ✅ | ✅ | ❌ |
| Ручной sync | ✅ | ✅ | ✅ | ❌ |
| AI-книги | ✅ | ✅ | ✅ | ❌ |
| Чтение документов | ✅ | ✅ | ✅ | ✅ |
| Доступ к /admin | ✅ | ✅ | ❌ | ❌ |

## Роли в пространстве

Назначаются через **Пространства** → редактирование → **Участники** / **Гroups**.

| Роль | Описание |
|------|----------|
| `admin` | Управление участниками пространства |
| `editor` | Создание и редактирование документов |
| `viewer` | Только чтение |

### Назначение прав

Два способа:

1. **Прямое членство** — пользователь добавлен в пространство с ролью
2. **Через группу** — пользователь входит в группу, группа добавлена в пространство

Эффективная роль — максимальная из всех назначений (прямых и через группы).

## Публичные пространства

Пространство с флагом **Публичный** (`is_public: true`):

- Документы доступны без авторизации
- Поиск по публичным документам — без входа
- Редактирование — только для авторизованных с ролью editor+

## OIDC: маппинг ролей

При входе через OIDC роли и группы синхронизируются из JWT claims:

| Claim | По умолчанию | Назначение |
|-------|-------------|------------|
| Role claim | `roles` | Системные роли |
| Group claim | `groups` | Группы пользователя |

Настраивается в **OIDC-провайдеры** → `role_claim`, `group_claim`, `sync_groups`.

## Ограничения admin при управлении пользователями

Пользователь с ролью `admin` (не super_admin):

- ✅ Может создавать/редактировать пользователей с ролями `viewer`, `editor`
- ❌ Не может управлять пользователями с ролями `admin`, `super_admin`
- ❌ Не может назначать роли `admin`, `super_admin`
- ❌ Не может создавать новых пользователей (только super_admin)

## Рекомендации

1. Минимизируйте количество `super_admin` (1–2 человека)
2. Используйте группы для массового назначения прав
3. Публичные пространства — только для документации без секретов
4. Синхронизируйте группы из OIDC (`sync_groups: true`)
