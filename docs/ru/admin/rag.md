# RAG индексация

**URL:** `/admin/rag`

> Требуется `admin` или `super_admin`.

## Назначение

RAG (Retrieval-Augmented Generation) разбивает текст документов на chunks и строит embeddings, чтобы авторизованные пользователи могли **спрашивать документацию** на странице поиска (`/search` → вкладка AI).

## Панель

| Метрика | Описание |
|---------|----------|
| Документов с chunks | Страницы, подготовленные к индексации |
| Текстовых chunks | Всего в `document_chunks` |
| Chunks с embeddings | Готовы к семантическому поиску |
| Chunks без embeddings | Ожидают воркер embeddings |

**Переиндексировать всё** — полная пересборка после массового импорта или смены настроек LLM.

## Требования

- `LLM_ENABLED=true` на `backend-server` для embeddings и `/api/rag/ask`
- Документы должны быть доступны пользователю, задающему вопрос (те же ACL, что у поиска)
