# Руководство администратора

Панель администратора доступна пользователям с системными ролями `admin` или `super_admin`.

**URL:** `/admin`

## Разделы панели

| Раздел | URL | Мин. роль |
|--------|-----|-----------|
| [Пространства](spaces.md) | `/admin/spaces` | admin |
| [Репозитории](repositories.md) | `/admin/repositories` | admin |
| [Пользователи](users.md) | `/admin/users` | admin (ограничено) |
| [Группы](groups.md) | `/admin/groups` | admin |
| [Настройки системы](settings.md) | `/admin/settings` | admin (запись: super_admin) |
| [OIDC-провайдеры](oidc.md) | `/admin/oidc` | super_admin |
| Аналитика | `/admin/analytics` | admin |
| [RAG индексация](rag.md) | `/admin/rag` | admin |
| Аудит | `/admin/audit` | super_admin |

## Системные роли

| Роль | Возможности |
|------|------------|
| `super_admin` | Полный доступ: OIDC, системные настройки, все пользователи |
| `admin` | Пространства, репозитории, группы, ограниченное управление пользователями |
| `editor` | Редактирование документов, sync, книги (без доступа к /admin) |
| `viewer` | Только чтение |

Подробнее: [RBAC](rbac.md)

## Типичный workflow администратора

```
1. Настроить OIDC и системные параметры
2. Создать пространства
3. Подключить Git-репозитории
4. Запустить синхронизацию
5. Создать группы и назначить права
6. Пригласить пользователей (или дождаться OIDC-provisioning)
```

## Ограничения для admin (не super_admin)

| Действие | admin | super_admin |
|----------|-------|-------------|
| Сохранение системных настроек | ❌ (только просмотр) | ✅ |
| Управление OIDC | ❌ | ✅ |
| Создание пользователей | ❌ | ✅ |
| Назначение super_admin/admin | ❌ | ✅ |
| Управление viewer/editor | ✅ | ✅ |
| Пространства и репозитории | ✅ | ✅ |

## Аудит

При включённом audit log (`enableAuditLog: true` в Helm) действия администраторов записываются в таблицу `audit_log`.
