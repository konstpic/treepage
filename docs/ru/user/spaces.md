# Пространства

**Пространство (Space)** — логический контейнер для коллекции документации. Каждое пространство может быть связано с одним или несколькими Git-репозиториями.

## Просмотр пространств

Перейдите в **Пространства** (`/spaces`).

### Без авторизации

Отображаются только **публичные** пространства.

### После входа

Отображаются все пространства, к которым у вас есть доступ (прямой или через группу).

## Карточка пространства

| Поле | Описание |
|------|----------|
| Название | Человекочитаемое имя |
| Slug | URL-идентификатор |
| Описание | Краткое описание содержимого |
| Доступ | Публичный / Приватный |

## Открытие пространства

Нажмите на пространство → откроется `/spaces/{slug}`.

### Что вы увидите

- Дерево документов в боковой панели
- Список привязанных репозиториев
- Кнопку синхронизации (если есть права editor+)
- Вкладку **Книги**

## Поиск по пространствам

На странице `/spaces` есть строка поиска — фильтрует по названию и описанию.

## Права доступа

| Тип пространства | Кто видит |
|-----------------|-----------|
| Публичное | Все (включая неавторизованных) |
| Приватное | Только участники и участники групп |

### Роли в пространстве

| Роль | Возможности |
|------|------------|
| `viewer` | Чтение документов и книг |
| `editor` | Чтение + редактирование + sync + создание книг |
| `admin` | Всё выше + управление участниками пространства |

## Если пространств нет

- **Без входа:** «Публичных пространств пока нет»
- **После входа:** «Пространств пока нет. Создайте в панели администратора»

Обратитесь к администратору для создания пространства.
