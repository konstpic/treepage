# Поиск

TreePage поддерживает **полнотекстовый поиск** и **RAG** — ответы по документации с помощью LLM.

Страница: `/search`

---

## Обычный поиск

### Как искать

1. Откройте **Поиск** в навигации
2. Введите запрос на вкладке поиска
3. Нажмите **Искать** или Enter

### Прямая ссылка

```
/search?q=kubernetes&space=engineering&tags=helm,deployment
```

### Поля поиска

| Поле | Описание |
|------|----------|
| Заголовок | H1 и title документа |
| Содержимое | Текст Markdown |
| Теги | Frontmatter `tags:` |
| Репозиторий | Имя Git-репозитория |
| Автор | Автор последнего изменения |

### Multilingual FTS (фаза Search & RAG)

Поиск работает на **русском и английском** (конфигурации PostgreSQL `russian`, `english`, `simple`).

### Фильтры

| Фильтр | Описание |
|--------|----------|
| Пространство | Ограничить одним space |
| Автор | Email автора |
| Теги | Через запятую |

### Лимиты

| Параметр | По умолчанию |
|----------|--------------|
| Лимит результатов | 20 |
| Максимум | 100 |

Настраивается в **Настройки системы** → **Платформа**.

## RAG — «Спросить документацию»

> Требуется настроенный LLM на `backend-server` (`LLM_ENABLED=true`). Для гибридного поиска — embeddings (`EMBEDDING_ENABLED=true`).

### Как пользоваться

1. Перейдите на `/search`
2. Введите вопрос на естественном языке (RU или EN)
3. Нажмите **Спросить документацию**

### Что возвращает система

| Элемент | Описание |
|---------|----------|
| **Ответ** | Краткий текст на языке вопроса, только по найденным фрагментам |
| **Цитаты** | Точные выдержки из документов со ссылками |
| **Источники** | Список документов; первый помечен как наиболее релевантный |
| **Уверенность** | При низкой confidence — предложения уточняющих вопросов |
| **👍 / 👎** | Обратная связь улучшает поиск (learned synonyms) |

### Как это работает (кратко)

```
Вопрос → FTS + keywords + vector (если embeddings)
      → фильтр RBAC + Page ACL
      → LLM формирует ответ
      → sources + citations
```

Подробнее: [Дорожная карта — Search & RAG](../reference/roadmap.md).

### API

```http
POST /api/rag/ask
Content-Type: application/json

{ "question": "Как развернуть локально?" }
```

```http
POST /api/rag/feedback
{ "question": "...", "helpful": true, "answer": "...", "sources": [...] }
```

---

## Доступ

- **Публичные** пространства — поиск без входа
- **Приватные** — после авторизации, с учётом RBAC и Page ACL (фаза 3)
- RAG использует те же правила доступа, что и обычный поиск

---

## Советы

- Добавляйте `tags:` в frontmatter для лучшей находимости
- Для RAG формулируйте конкретные вопросы: «Как настроить OIDC?», а не одно слово
- После `git pull` и sync дождитесь индексации chunks (автоматически при sync)
