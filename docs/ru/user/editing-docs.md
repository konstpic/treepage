# Редактирование документов

> Требуется роль **editor** или выше в пространстве (или системная роль `admin`/`super_admin`).

## Открытие редактора

1. Откройте документ
2. Нажмите **Редактировать**
3. Панель форматирования: заголовки, жирный, таблицы, tags, Mermaid, wiki-ссылки
4. **Ctrl+Space** — автодополнение (можно отключить в тулбаре)
5. Правый клик — контекстное меню форматирования
6. Переключайтесь между **Редактор** и **Просмотр**

## Сохранение и публикация

| Действие | Описание |
|----------|----------|
| **Сохранить локально** | Запись в PostgreSQL + новая версия (опубликовано) |
| **Сохранить черновик** | Сохранение без публикации — видно только редакторам |
| **Опубликовать** | Сделать страницу видимой для всех с доступом на чтение |
| **Отправить PR** | Для Git-linked страниц: ветка, commit, MR/PR (GitHub или GitLab) |

## Черновики, избранное, вложения

- Черновики отмечены в дереве документов
- Звёздочка на странице — **избранное** (раздел **Мои страницы**, `/me`)
- Недавно просмотренные — там же
- **Вложения** — загрузка файлов на странице документа (до 10 МиБ)
- **Уведомления** — колокольчик в шапке (публикации, новые документы)

## Git-backed документы и конфликты

Если страница синхронизирована из Git:

- Локальное сохранение помечает страницу как **имеющую неопубликованные правки**
- **Git sync пропускает** такие страницы (не перезаписывает) — в ответе sync будет `conflicts_skipped`
- Чтобы опубликовать upstream: **Отправить PR** или откатите правки через историю версий

## История версий

1. Откройте **История версий**
2. **Сравнить** — diff между версиями
3. **Восстановить** — откатить документ к выбранной версии (создаётся новая версия)

## Удаление страницы

Кнопка корзины в заголовке документа (только для editor+).
